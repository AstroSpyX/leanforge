EVAL CURLS — one per model, same prompt, copy/paste/run
=========================================================

PROMPT (used in every curl below)
  "Prove the following Lean 4 theorem:

  ```lean4
  import Mathlib

  theorem sqrt_two_irrational : Irrational (Real.sqrt 2) := by
    sorry
  ```

  Replace the sorry with a valid proof."

To change the prompt: edit the "content" field in the messages array
of each curl. All five curls use identical messages.

Sampling parameters: temperature=0.6, top_p=0.95, num_predict=1500.
Picked to match Kimina's trained defaults; the other models will also
accept these. Lower num_predict for faster runs, raise for more room.

-----------------
curl http://localhost:11434/api/chat -d '{

5b. deepseek-prover:q5_k_m   (PROPER format — model's trained wrapper)
-----------------------------------------------------------------------
No system message. User message uses the exact "Complete the
following Lean 4 code... proof plan" wrapper DeepSeek-Prover-V2
was fine-tuned on (per the HF model card).

IMPORTANT: must use sampling (temperature > 0). Greedy decoding
(temperature=0) sends this model into a degenerate repetition loop
that emits 1500 tokens of the same paragraph and never produces
Lean code. Confirmed during eval. The HF card omits sampling
params; defaulting to Kimina-style temperature=0.6, top_p=0.95.

curl http://localhost:11434/api/chat -d '{
  "model": "deepseek-prover:q5_k_m",
  "stream": false,
  "options": {"temperature": 0.6, "top_p": 0.95, "num_predict": 1500},
  "messages": [
    {"role": "user", "content": "Complete the following Lean 4 code:\n\n```lean4\nimport Mathlib\n\ntheorem sqrt_two_irrational : Irrational (Real.sqrt 2) := by\n  sorry\n```\n\nBefore producing the Lean 4 code to formally prove the given theorem, provide a detailed proof plan outlining the main proof steps and strategies.\nThe plan should highlight key ideas, intermediate lemmas, and proof structures that will guide the construction of the final formal proof."}
  ]
}'

Notes
-----
- Same prompt is sent to all five models. Prover models (kimina,
  deepseek) are NOT given their trained prompt format here. This is
  intentional — it's the naive baseline. Compare against per-model
  formatted versions later.
- qwen2-math has num_ctx=4096; the prompt here is short enough to fit.
  Longer prompts may need a smaller model or num_ctx override.
- deepseek-prover loads at ~22GB resident. If kimina is still loaded,
  one of them will be evicted before deepseek starts answering. Run
  `ollama ps` between calls if you want to confirm.
- First call to a model that isn't currently resident adds 5-20s
  cold-start. Look at load_duration in the response (0 means warm).
